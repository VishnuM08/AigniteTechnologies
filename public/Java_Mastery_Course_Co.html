Three terms beginners confuse — let's settle them for good.

JVM — Java Virtual Machine

The engine that actually runs your code
What it is
A virtual machine — a piece of software that pretends to be a computer. Java code doesn't run on your real CPU; it runs on the JVM, which runs on your CPU.
What it does
• Loads compiled .class files (bytecode)
• Verifies the bytecode is safe to run
• Translates bytecode to machine code (JIT)
• Manages memory — allocation and garbage collection
• Provides runtime services to your program
Key insight: JVM is platform-specific, but bytecode is not.

if / else / else if

The most basic decision structure
Basic syntax
if (score >= 90) {
    System.out.println("Grade: A");
} else if (score >= 80) {
    System.out.println("Grade: B");
} else if (score >= 70) {
    System.out.println("Grade: C");
} else {
    System.out.println("Grade: F");
}

// Java checks conditions top-to-bottom.
// First match wins. else runs if none match.
Ternary — one-line if/else
int max = (a > b) ? a : b;

String label = isActive
    ? "online"
    : "offline";
Tip: use braces { } even for one-line bodies. It prevents bugs when you add a second line later.
The condition inside () must be a boolean. Unlike C or JavaScript, Java will not accept if (1) or if (someString).

while & do-while

Loop until a condition fails
while — check first, then run
int i = 0;
while (i < 5) {
    System.out.println(i);
    i++;
}

// If condition is false at the start,
// the body never runs at all.
do-while — run first, then check
int i = 0;
do {
    System.out.println(i);
    i++;
} while (i < 5);

// Body always runs at least once,
// even if condition is false.
Which loop to use?
for — you know the number of iterations (counting through a range or array)
while — you don't know upfront; loop until some condition changes
do-while — same as while, but you need the body to run at least once

Controlling the flow

break & continue
Two keywords that change how loops behave from the inside.

break & continue

Two ways to alter a loop mid-iteration
break — exit the loop immediately
for (int i = 0; i < 10; i++) {
    if (i == 5) break;
    System.out.println(i);
}
// 0, 1, 2, 3, 4 — stops at 5
continue — skip to the next iteration
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) continue;
    System.out.println(i);
}
// 1, 3, 5, 7, 9 — only odd
Visual difference
break jumps OUT of the loop entirely. No more iterations.
continue skips the rest of the CURRENT iteration; the loop continues with the next one.

Method overloading

Same name — different parameter list — Java picks the right one
int add(int a, int b)         { return a + b; }
double add(double a, double b)   { return a + b; }
int add(int a, int b, int c)   { return a + b + c; }
String add(String a, String b)   { return a + b; }

// Java chooses based on the arguments
add(1, 2);              // calls int version → 3
add(1.5, 2.5);          // calls double version → 4.0
add(1, 2, 3);           // calls 3-arg version → 6
add("Hi ", "Java");    // calls String version → "Hi Java"
✓ Allowed (different parameters)
• Different number of parameters
• Different types of parameters
• Different order of types
✗ Not allowed
• Only return type differs
• Only parameter names differ
• Java needs to tell calls apart by arguments

Exception best practices

Common mistakes and how to avoid them
❌ Don't swallow exceptions
Empty catch blocks hide real problems and make debugging nightmares.
catch (Exception e) {
    // nothing — BAD
}
✓ Be specific
Catch the narrowest exception you can handle. Avoid catch (Exception e).

✓ Exceptions for exceptional cases
Don't use exceptions for normal control flow. They're slow and obscure intent.
// BAD: parse failure as flow control

The PECS rule

Producer Extends, Consumer Super
"Producer Extends, Consumer Super"

Producer → ? extends

If a collection produces values (you read from it), use ? extends T. You can take values OUT (typed as T), but can't put values IN.

void copy(List<? extends T> src, …)

Consumer → ? super

If a collection consumes values (you write to it), use ? super T. You can put values IN (of type T), but can't safely take typed values OUT.

void copy(…, List<? super T> dst)
Standard library example: Collections.copy(List<? super T> dst, List<? extends T> src)

Transactions

Group multiple changes — all-or-nothing
try (Connection conn = DriverManager.getConnection(url, u, p)) {
    conn.setAutoCommit(false);   // switch off auto-commit

    try {
        // Withdraw from one account...
        try (PreparedStatement ps = conn.prepareStatement(
             "UPDATE accounts SET balance = balance - ? WHERE id = ?")) {
            ps.setDouble(1, 100); ps.setInt(2, 1);
            ps.executeUpdate();
        }
        // ...deposit to another
        try (PreparedStatement ps = conn.prepareStatement(
             "UPDATE accounts SET balance = balance + ? WHERE id = ?")) {
            ps.setDouble(1, 100); ps.setInt(2, 2);
            ps.executeUpdate();
        }
        conn.commit();              // both succeeded → save
    } catch (SQLException e) {
        conn.rollback();            // any failure → undo all
        throw e;
    }
}

Sealed classes (Java 17+)

Restricted inheritance — for tighter modeling
// Only these three can extend Shape — nothing else.
public sealed interface Shape permits Circle, Square, Triangle {}

public record Circle(double radius)    implements Shape {}
public record Square(double side)      implements Shape {}
public record Triangle(double b, double h) implements Shape {}

// === Why this is powerful ===
// The compiler KNOWS there are exactly 3 implementations.
// So a switch on Shape is exhaustive — no default needed:

double area = switch (s) {
    case Circle c   -> Math.PI * c.radius() * c.radius();
    case Square sq  -> sq.side() * sq.side();
    case Triangle t -> 0.5 * t.b() * t.h();
    // no default — compiler knows we covered everything
};
